# Copy-Hunting Checklist for DineSmart-AI

Use this checklist to look for public copies of your code, UI text, branding, or deployment.

## 1) Save evidence first
Before changing the repo further, save:
- Screenshot of GitHub Traffic page
- Repo URL
- Current default branch name
- Latest commit hash
- Dates when the repo was public
- Screenshots of any sensitive files that were exposed
- Screenshots of your live app and major pages

## 8) What counts as strong evidence
A copy claim gets stronger if you find multiple matches like:
- Same domain references
- Same unusual text or typos
- Same component names
- Same slugs or IDs
- Same prompt wording
- Same emails
- Same test/demo users
- Same file/folder structure

## 9) Keep a simple evidence log
For each suspected copy, record:
- Date found
- URL
- Screenshot
- Matching strings
- Matching files/lines
- Why it appears copied

Suggested format:
- Suspect URL:
- Found on:
- Matching evidence:
- Screenshots saved at:
- Notes:

## 10) Important limitation
Traffic/clones prove exposure, not identity.
You may be able to show your repo was widely cloned, but not exactly who copied it.
